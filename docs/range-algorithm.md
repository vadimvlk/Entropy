## 1. Суть в одном абзаце

📝 Рынок ходит «диапазонами». Диапазон рождается так: цена делает **хай** `H`, откатывает к **низу** `L`, затем **повторно пробивает** `H` (перехай). Величина диапазона `D = H − L`. В момент перехая возникают **сразу две цели**: уйти вверх на величину диапазона (`H + D`) или вернуться к нижней границе (`L`). Исход случаен, но состояний всего два — «идём вверх» или «идём вниз». Состояние определяется **контекстным уровнем** (локальный низ «пилы»), а не объёмами/фибо/фигурами. При достижении границы и её пробое рождается новый диапазон — и так рекурсивно.

---

## 2. Базовые понятия

| Понятие | Определение | Источник |
|---|---|---|
| **Swing high / low** | Локальный экстремум цены (пивот) | 📝 (ты оперируешь хаями и низами) |
| **Порог свинга `θ`** | Минимальный откат (в %, или в ATR), чтобы движение считалось разворотом, а не шумом | ➕ **МОЁ** — ты порог не задавал, но без него «пивот» не определить на тиковом шуме |
| **Хай диапазона `H`** | Цена первого экстремума (в примере `100`) | 📝 |
| **Низ диапазона `L`** | Низ отката после `H` (в примере `80`) | 📝 |
| **Величина `D`** | `D = H − L` (в примере `20`) | 📝 |
| **Перехай / активация** | Повторный пробой `H` после отката к `L` | 📝 |
| **Верхняя цель `TP↑`** | `H + D` (в примере `120`) | 📝 |
| **Нижняя цель** | возврат к `L` (в примере `80`) | 📝 |
| **Контекстный уровень `CL`** | Локальный низ «пилы» после пробоя — пивот, который определяет состояние (в примере `99`) | 📝 |

➕ **МОЁ (предложение):** величину диапазона считать можно двумя способами:
- арифметически `D = H − L` (твоя модель, $20), **или**
- в логарифмах/процентах `d = ln(H/L)`.
На геометрическом графике (а движок именно такой) лог-версия чуть корректнее, но для твоей интуиции оставлю **арифметику по умолчанию**, лог — как переключатель. ❓ Подтверди, что ок считать арифметически.

## 3. Состояния автомата (FSM)

➕ **МОЁ — это моя структуризация твоей логики в конечный автомат.** Сами правила переходов — твои, имена состояний придумал я.

```
            перехай H (после отката к L)
 SEEKING ───────────────────────────────►  BREAKOUT_UP
   ▲                                          │
   │ (новый диапазон)                         │ ныряет под H к локальному низу,
   │                                          │ затем новый хай  → фиксируем CL
   │                                          ▼
   │                                      CONTEXT_BULL  (состояние: BUY → TP↑)
   │                                          │
   │                  цена > CL: штурм H..    │  пробой CL (CL − ε)
   │                  держим BUY → TP↑        ▼
   │                                      CONTEXT_BEAR  (состояние: SELL → L)
   │                                          │
   │            достигли L ───────────────────┘
   │                  │
   │                  ├─ отбой от L  → новый диапазон [L, H_недавний], идём к H_недавнему
   │                  └─ пробой L    → measured move к L − D'  (D' нового диапазона)
   └──────────────────┘
```

### Состояния и переходы

| Состояние | Что значит | Переход дальше |
|---|---|---|
| **SEEKING** | Активного диапазона нет. Ищем структуру: хай `H` → откат ≥ `θ` к `L` → перехай `H`. | На перехае `H` → `BREAKOUT_UP`. |
| **BREAKOUT_UP** | 📝 Цена вышла за `H`. Возможны три сценария (см. §4). | Безоткатно к `TP↑`; либо сразу назад в диапазон и к `L`; либо «пила» → формируется `CL` → `CONTEXT_BULL`. |
| **CONTEXT_BULL** | 📝 Состояние **BUY → TP↑**, пока `цена > CL`. | Цена снова берёт зону хая → подтверждаем BUY. Пробой `CL` → `CONTEXT_BEAR`. |
| **CONTEXT_BEAR** | 📝 Состояние **SELL → L** (контекст пробит вниз). | Достигли `L` → развилка (отбой / пробой). |
| **(в `L`)** | 📝 Достигли нижней границы. | Отбой → новый диапазон `[L, H_недавний]` (идём к `H_недавнему`). Пробой `L` → measured move `L − D'`. |

❓ **ДОДУМАЛ — проверь:**
1. **Чистый безоткатный пробой** (цена ушла на `TP↑` без «пилы»): я считаю, что состояние сразу = **BUY → TP↑**, стоп под `L` (или под точкой пробоя). Верно? Или до появления `CL` ты считаешь состояние неопределённым и не входишь?
2. **«Тут же вернулась в диапазон и рухнула к `L`»** (второй сценарий): тут `CL` так и не сформировался. Я трактую это как: если после пробоя `H` цена возвращается **под `H`** и не делает новый хай, а идёт к `L` — это сразу медвежий сценарий (равносильно `CONTEXT_BEAR` без `CL`). Верно?

## 4. Три сценария после пробоя `H` (твой ключевой кусок)

📝 Дословно по твоему описанию, формализовано:

**Сценарий 1 — безоткатный вынос:** `H → TP↑` без возврата. (100 → 120 напрямую.)

**Сценарий 2 — ложный пробой:** цена вышла за `H`, тут же вернулась в диапазон и пошла к `L`. (100 → чуть выше → 80.)

**Сценарий 3 — «пила» и контекстный уровень:**
1. Пробой: `100 → 105` (локальный хай выше `H`).
2. Откат «пилы»: `105 → 99` — **ниже `H`**. Этот низ `99` = **контекстный уровень `CL`**.
3. Новый хай: `99 → 106` (выше прошлых `105`) — **подтверждает `CL`**.
4. Состояние теперь **BUY → TP↑** пока `цена > 99`.
5. 📝 «Часто с контекстных уровней цена опять штурмует `106-107` → к цели `120`; либо снова к `CL`, но уже тестирует `98.9`».
6. 📝 «Бывает, `CL` в итоге пробивается → состояние вниз → цена летит к `L (80)`».

❓ **ДОДУМАЛ — определение `CL` точно:** `CL` = локальный минимум («higher-low»), который рынок **уважил** по пути к новому хаю. То есть фиксируется, когда после провала к локальному низу цена делает **новый хай выше предыдущего локального хая**. В примере: низ `99` подтверждается, когда взяли `106 > 105`. Верно так, или `CL` фиксируется уже при любом возврате выше `H`?

## 5. Рекурсия вниз (новый диапазон у границы)

📝 Дословно: «при достижении `80` уже есть новый диапазон — `80` и тот хай, с которого только что ушли вниз (`106`). И снова развилка: отбились в диапазоне → идём на `106`; пробили `80` → уходим на величину».

Формализация:
- Достигли `L = 80`. Берём **последний значимый хай перед сливом** `H_new = 106`.
- Новый диапазон `[80, 106]`, новая величина `D' = 106 − 80 = 26`.
- Развилка у `80`:
  - **Отбой** → цель вверх `H_new = 106` (откат внутри диапазона).
  - **Пробой `80`** → measured move вниз: `80 − D' = 80 − 26 = 54`.

❓ **ДОДУМАЛ — проверь:** при пробое вниз measured move = `L − D'`, где `D'` — величина **нового** диапазона `[L, H_new]` (=26), а не старого `D` (=20). Так? (Логично, т.к. размах изменился.)

➕ **МОЁ:** это полностью симметричная конструкция — всё, что описано «вверх», зеркально работает «вниз». То есть автомат один и тот же, просто стороны меняются. Для шорт-сценария: низ `L` играет роль «хая наоборот», `CL` — локальный «lower-high» и т.д.

## 6. Точки решения: вход / стоп / тейк

➕ **МОЁ — это параметры, которые ты сам предложил перебирать («ждать 50% движения или пробой»).** Это НЕ часть твоей жёсткой логики, а настраиваемые «ручки», которые будем тюнить в бэктесте.

| Ручка | Смысл | Кандидаты для перебора |
|---|---|---|
| `entryMode` | момент входа | на пробое `H` / на ретесте `H` после пробоя / после прохождения `entryFrac` пути к цели |
| `entryFrac` | сколько % хода к `TP↑` ждём перед входом | 0% / 25% / **50%** / 75% |
| `stopMode` | где стоп | граница диапазона (`L`) / контекстный уровень `CL` |
| `tpMode` | где тейк | полный `H+D` / частичный (доля `D`) / трейл по последовательным `CL` |
| `flipMode` | переворачиваемся ли при пробое `CL`/`L` | да (sell сразу) / нет (только закрытие) |
| `sizeFrac` | размер позиции / плечо | 0.25x / 0.5x / 1x / макс |
| `tf` | таймфрейм, на котором ищем свинги | 1s / 5s / 15s |

❓ Здесь жду твоих указаний: какие из этих ручек фиксировать жёстко (часть твоей идеи), а какие реально перебирать.

## 7. ➕ Математические нюансы (всё в этом разделе — МОЁ)

> Это то, что я добавляю «от математики». Это **предсказания**, которые бэктест проверит на данных.

### 7.1 Вероятность «вверх раньше вниз» (Gambler's Ruin / optional stopping)
Цена в движке — геометрическое броуновское движение **без сноса**, значит **лог-цена — мартингал**. Для старта из точки пробоя `H` между барьерами `L` (низ) и `TP↑` (верх):

```
P(достигли TP↑ раньше, чем L) = ln(H/L) / ln(TP↑/L)
```

Для `80 / 100 / 120`: `ln(100/80) / ln(120/80) = 0.2231 / 0.4055 ≈ 0.550`.

- Наивная арифметика дала бы ровно **50/50**.
- «Лишние» ~5% — это **чистая геометрия лог-шкалы** (верхняя цель ближе в лог-мере), **а не** предсказание из паттерна диапазона.

### 7.2 Матожидание сделки
При симметричных целях ±`D` (тейк `+D`, стоп `−D`) и `p ≈ 0.55`:

```
EV = D·(2p − 1) ≈ 20 · 0.10 = +2.0   (на единицу размера)
```

Положительное — **но** это ровно **снос buy&hold** (выпуклость, эффект Йенсена) за время удержания. **Случайный** вход лонгом за то же время даёт столько же. Где именно стоят уровни `80/100/120` — **роли не играет**. Шорт даёт симметричный **−EV**.

### 7.3 Плечо и volatility drag
Геометрический рост капитала при плече `k`: `g(k) ≈ (σ²/2)·k·(1 − k)`.
- `k = 1` → `g = 0` (ноль роста).
- `k > 1` → `g < 0` (плечо **съедает** капитал даже при +EV в матожидании).
- Оптимум ≈ `k = 0.5` (дробное плечо по Келли).
- При `k = 10` (макс. плечо движка) на диапазоне в десятки процентов тебя, скорее всего, **ликвиднёт до достижения цели** — это честно моделируется `Account.liquidateIfNeeded`.

### 7.4 Что это значит для проверки
**Killer-эксперимент:** сравниваем твою range-стратегию с baseline'ами:
- `buyHold` (всегда лонг),
- `randomEntry` (случайный вход с тем же распределением TP/SL),
- `flat` (ничего не делаем).

Ожидаемый результат: `range-long ≈ randomEntry-long ≈ buyHold`, а `range-short ≈ −(то же)`, всё в пределах 95% доверительного интервала. Это **доказывает данными**, что геометрия диапазонов не даёт направленного преимущества. Если range устойчиво обгонит randomEntry — копаем (реальный паттерн или баг).
