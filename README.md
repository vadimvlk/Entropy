# ENTROPY — Random Walk Terminal

Веб-приложение «случайная величина»: непрерывный поток случайных тиков (как сделки),
из которых строятся японские свечи OHLC на лестнице таймфреймов — в стиле TradingView, с
торговлей, объёмом и инструментами. Цена движется как криптоактив (всегда положительна, может
стремиться к нулю).

**Теперь поток живёт на сервере и идёт 24/7.** Включил один раз — данные генерируются непрерывно
(и пока сервер работает, и переживая перезапуски/краши через SQLite), независимо от того, открыт ли
где-то браузер. Сбрасывается только вручную. Браузер — тонкий клиент: тянет историю по HTTP и живые
бары по SSE.

Стек: **клиент** — Vite + TypeScript + TradingView Lightweight Charts; **сервер** — Node
(`node:http` + встроенный `node:sqlite`, без нативных зависимостей).

## Запуск

### Разработка (два процесса)

```bash
npm install
npm run dev:server   # бэкенд-поток на http://127.0.0.1:8787 (генерит данные, пишет в SQLite)
npm run dev          # клиент с HMR на http://127.0.0.1:5173 (проксирует /api → 8787)
```

Откройте `http://127.0.0.1:5173`.

### Продакшн (один процесс)

```bash
npm run build        # tsc + сборка клиента в dist/
npm start            # сервер на :8787 раздаёт dist/ и API/SSE
```

Откройте `http://127.0.0.1:8787`.

### Тесты

```bash
npm run test:unit    # ядро: агрегация, эквивалентность роллапов = aggregate(), round-trip SQLite, торговля, фибо
```

### Переменные окружения сервера

| Переменная | По умолчанию | Назначение |
|-----------|--------------|-----------|
| `PORT` | `8787` | Порт HTTP/SSE |
| `DB_PATH` | `./data/market.db` | Файл SQLite с историей и состоянием |
| `CONTROL_TOKEN` | *(авто-генерится)* | Секрет для управляющих действий (см. ниже) |

## Токен управления

Просмотр графика **открыт** (без токена). Управляющие действия — **сброс**, режим
**Маркет-Мейкер** и его толчки, скорость/режим генератора, пауза/старт — требуют **токена**.

### Как получить токен

- **Не задавали `CONTROL_TOKEN`** → сервер генерирует случайный токен при старте и печатает его в
  консоль:

  ```
  ┌─ CONTROL TOKEN (auto-generated; set CONTROL_TOKEN to pin it) ─
  │  Xq9b2c...                                                    (ваш токен)
  └──────────────────────────────────────────────────────────────
  ```

- **Хотите постоянный токен** (рекомендуется для VPS) → задайте свой:

  ```bash
  CONTROL_TOKEN="$(openssl rand -base64 24)" npm start      # сгенерировать и зафиксировать
  # или просто: CONTROL_TOKEN="мой-секрет" npm start
  ```

  В systemd-юните (см. ниже) токен задаётся через `Environment=CONTROL_TOKEN=…`.

### Как использовать токен

В интерфейсе нажмите кнопку-**замок** 🔒 (в правой группе шапки) → вставьте токен. Он проверяется
на сервере (`POST /api/control/verify`), сохраняется в `localStorage` браузера и автоматически
прикладывается ко всем управляющим запросам (`Authorization: Bearer …`). Замок загорается —
управление разблокировано. Повторный клик по замку — заблокировать (забыть токен).

> Токен защищает только управление. На публичном VPS дополнительно используйте HTTPS-прокси
> (см. деплой). Сравнение токена — constant-time.

## Деплой на VPS

```bash
# на сервере
git clone … && cd App
npm ci
npm run build

# systemd-юнит /etc/systemd/system/entropy.service
# [Service]
#   WorkingDirectory=/opt/entropy
#   Environment=PORT=8787
#   Environment=CONTROL_TOKEN=ваш-длинный-секрет
#   Environment=DB_PATH=/var/lib/entropy/market.db
#   ExecStart=/usr/bin/npm start
#   Restart=always
sudo systemctl enable --now entropy
```

Перед ним — обратный прокси с TLS. Caddy (авто-сертификат) — проще всего:

```caddy
entropy.example.com {
    reverse_proxy 127.0.0.1:8787
}
```

> Для SSE важно отключить буферизацию у прокси. Сервер уже шлёт `X-Accel-Buffering: no` (nginx);
> Caddy и так стримит без буфера. Требуется Node ≥ 24 (стабильный `node:sqlite`).

## Docker

```bash
# с авто-генерацией токена (смотрите его в логах)
docker compose up -d --build
docker compose logs entropy        # здесь будет напечатан CONTROL TOKEN

# или с зафиксированным токеном
CONTROL_TOKEN="$(openssl rand -base64 24)" docker compose up -d --build
```

Откройте `http://<host>:8787`.

- **Образ multi-stage**: стадия сборки делает `dist/` (клиент) и бандлит сервер (+ общее ядро)
  в один файл `dist-server/index.mjs`; рантайм-образ — на `node:24-alpine` **без `node_modules`**
  (сервер использует только встроенные модули, включая `node:sqlite`).
- **Персистентность**: SQLite лежит на именованном томе `entropy-data` (`/data/market.db`), поэтому
  поток продолжается без разрыва при `restart` и пересоздании контейнера. `restart: unless-stopped`
  держит его «всегда включённым».
- **Healthcheck** дергает `/api/health`. `CONTROL_TOKEN` задаётся через окружение (или авто-генерится).

Только образ, без compose:

```bash
docker build -t entropy .
docker run -d --name entropy -p 8787:8787 \
  -e CONTROL_TOKEN=ваш-секрет \
  -v entropy-data:/data \
  --restart unless-stopped entropy
```

За контейнером — тот же обратный прокси с TLS (Caddy/nginx), что и выше.

## Возможности

### Поток и таймфреймы
- **Реальное время**: 1 секунда симуляции ≈ 1 реальная секунда. Поэтому 1м-свеча = реальная минута,
  1ч = реальный час, дневная свеча набирается за реальные сутки — большие ТФ наполняются по мере того,
  как сервер работает (для того и нужна персистентность).
- **Лестница ТФ**: секундные `1с / 5с / 15с` (в выпадающем списке **СЕК**) + `1м, 5м, 15м, 30м, 1ч,
  4ч, 12ч, 1д` (кнопки). Сервер ведёт **отдельную серию роллапов на каждый ТФ** и отдаёт готовые бары.
- **Поток тиков со случайной скоростью** — режимы `burst / fast / steady / slow / sparse`; в **АВТО**
  меняются сами, в **РУЧ** задаются вручную (управление — по токену, влияет на общий поток).
- **Модель цены** — мультипликативное случайное блуждание (как крипта): всегда > 0, без дрейфа,
  минимум `1e-7`. Волатильность: `quiet / normal / active / wild`.
- Свечи / линия / область; **объём** отдельной панелью (кнопка **Vol**); метки **High/Low** видимого
  участка; светлая/тёмная темы; зум/панорама.

### Торговля (локальная, в браузере)
- Рыночные **Buy / Sell**, плечо **1:10**, шаг/мин **0.1**, депозит **$1000**. Торговля идёт против
  общей серверной цены, но счёт у каждого браузера **свой** (в `localStorage`) — токен не нужен.
- Нетто-позиция с разворотом, средняя цена входа, реализованный/нереализованный P&L, маржа,
  **ликвидация** при equity ≤ 50% маржи. Линии входа/ликвидации и маркеры сделок на графике.

### Режим Маркет-Мейкер (по токену)
- Тумблер **MM** переворачивает панель (3D-флип). Кнопки **ВВЕРХ/ВНИЗ** толкают **общую** цену на
  ±$воздействие (отдельный тик поверх случайного потока) — действие глобальное, поэтому требует токена.

### Инструменты
- Перекрестие/магнит, горизонтальная/вертикальная линии, **Фибоначчи 0/50/100/200%** (тянется вправо),
  стирание/очистка. Линии, фибо и вид сохраняются в браузере.

### Сброс
Кнопка сброса (по токену) стирает серверную историю и начинает новую случайную последовательность —
**для всех зрителей**: сервер рассылает событие `reset` по SSE, клиенты перечитывают серию и
обнуляют свой локальный счёт.

## Архитектура

Поток живёт на сервере; браузер — тонкий клиент. Общее ядро (`shared/`) запускается и в Node, и в браузере.

| Файл | Назначение |
|------|-----------|
| `shared/engine.ts` | Генератор тиков (GBM), режимы авто/ручной, сериализация состояния (без БД/DOM) |
| `shared/candles.ts` | Модель свечи, лестница ТФ, лимиты ретеншена, `aggregate` / `lastGroup` |
| `shared/regimes.ts` | Режимы скорости и волатильности |
| `shared/rng.ts` | Гауссов сэмплер и помощники |
| `server/index.ts` | HTTP + SSE + раздача `dist/` + токен-аутентификация + раннер тиков |
| `server/seriesStore.ts` | Инкрементальные роллапы по всем ТФ + ретеншен + флаш в БД |
| `server/db.ts` | SQLite (`node:sqlite`): схема свечей/мета, батч-запись, prune, load |
| `src/marketClient.ts` | Клиент: история по HTTP, живой поток по SSE, управление по токену |
| `src/chart.ts` | Контроллер Lightweight Charts: серии, объём, темы, инструменты, позиция, High/Low, зум |
| `src/trading.ts` | Локальный торговый счёт: неттинг с разворотом, P&L, маржа, ликвидация |
| `src/main.ts` | Связывание клиента потока, графика, торговли и UI |
| `src/storage.ts` | Локальная персистентность браузера: рисунки, вид/тема, счёт, токен |
| `src/fib.ts`, `src/hilo.ts`, `src/verticalLine.ts`, `src/format.ts` | Примитивы графика и форматирование |

### Хранение и ретеншен
SQLite на сервере (`DB_PATH`). 1-секундная серия хранит **≥ 24 часа** (86 400 свечей); каждый старший
ТФ капится до 20 000 баров — на дневном ТФ это десятилетия истории при общем размере в единицы МБ.
Запись батчами раз в ~2.5 с + финальный флаш при остановке. Простой сервера не «доливается»: после
рестарта поток **бесшовно продолжается** с сохранённого состояния (без разрыва в `time` свечей).

### HTTP/SSE API

| Метод | Путь | Доступ | Назначение |
|-------|------|--------|-----------|
| `GET` | `/api/series?tf=&limit=` | открыто | История баров одного ТФ (компактные массивы) |
| `GET` | `/api/stream?tf=` | открыто | SSE: живой бар + статус хедера; события `tick` / `running` / `reset` |
| `POST` | `/api/control/verify` | токен | Проверка токена |
| `POST` | `/api/control/{reset,nudge,pause,resume,mode,regime}` | токен | Управление общим потоком |
